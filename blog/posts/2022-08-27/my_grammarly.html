Grammarly Latex语法检查工具

2022-08-27

演示界面：https://huacishu.com/app/grammarly/ 源代码：Github 下载到本地使用：点击此处

这是做什么用的？

Latex中的一些符号（如公式$$、引用~\cite）会干扰Grammarly的语法检查， 例如这个句子中modalitiies拼写错误，但Grammarly无法检测出来:

为此我写了一个小工具，配合Grammarly的浏览器插件使用，去除latex中的符号以方便语法检查，检查完毕后再转换回去：

现有功能

去除注释、引用（~\cite、~\shortcite、 ~\ref、~\cref）与行内公式$$

例如对于如下内容： %Some comments Bain~\shortcite{358_frozen} used a space-time transformer to % Inline comments Given an unlabeled video dataset $\mathcal{D}=\{x_i\}_{i=1}^N$, where each instance $x_i$ has two modalitiies: voice $x^v_i$ and face $x^f_i$.

点击“转换”会符号转变为编号[*]形式： Bain[0] used a space-time transformer to Given an unlabeled video dataset [1], where each instance [2] has two modalities: voice [3] and face [4].

再点击“回写”则将编号替换回原内容:

Bain~\shortcite{358_frozen} used a space-time transformer to Given an unlabeled video dataset $\mathcal{D}=\{x_i\}_{i=1}^N$, where each instance $x_i$ has two modalities: voice $x^v_i$ and face $x^f_i$.

本地部署

  • 演示页面无法保证一直可用，所以推荐下载到本地，用浏览器打开index.html就可以
  • 转换由浏览器本地实现，不会向服务器发送内容，主要功能实现在js/latex_util.js，可以根据自身的需要补充功能
  • 需求是众口难调的，所以有生之年不会有更新

相关解决方案

  • Overleaf平台有个Chrome插件：Overleaf textarea 特点：功能强大，也更具破坏性

  • VSCode插件：Grammarly for VS Code

  • 命令行方式将.tex文件转换为文本（不可逆） pandoc: pandoc --to=plain --wrap=none your_file.tex detex: detex your_file.tex > your_file.txt
